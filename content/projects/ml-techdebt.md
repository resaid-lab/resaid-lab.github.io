## Overview

ML software has distinctive maintenance risks because data, models, pipelines, and
code evolve together. This project studies how technical debt appears in ML
repositories, how data-science pipelines are structured in practice, and the
infrastructure needed to mine and analyze ML software at scale.

## Current Focus

- Taxonomies of self-admitted technical debt in ML codebases
- Empirical structure of data-science pipelines in theory and practice
- Repository-mining infrastructure for studying data-science software

The project groups papers on ML technical debt, data-science pipelines, and
large-scale mining of Python data-science software.
