## Overview

Coding agents are increasingly used for bug fixing, configuration, refactoring, and
software maintenance. Their failures are not limited to adversarial prompts: they
also occur during ordinary, goal-directed work when agents misunderstand constraints,
modify the environment destructively, or report success without completing the task.

## Current Focus

- Operational safety failures in deployed coding-agent ecosystems
- How TODO comments, prompts, and maintenance context shape generated code
- Benchmarks and taxonomies that reflect everyday software-development risk

This project connects recent work on agentic-code safety with earlier work on
AI-assisted technical-debt maintenance.
